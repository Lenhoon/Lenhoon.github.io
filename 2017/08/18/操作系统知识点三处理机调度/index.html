一、处理机调度

这一章主要是讲的几种调度算法

  1. 基本概念：进程数量>处理机数量，从就绪队列中按照一定的算法，选择一个进程并将处理机分配各它。
  2. 调度的层次：一个作业从提交开始直到完成，往往要经历以下三级调度
    ①作业调度：高级调度，从外存调度到内存中，以使他们获得竞争处理机的权利，本质就是内存与辅存之间的调度，频率低
    ②中级调度：内存调度，目的提高内存利用率和系统吞吐量，不能运行的进程，调至外存等待，进程状态为挂起状态
    ③进程调度：低级调度
  3. 不能进行调度的情况：
    ①中断过程
    ②进程在操作系统内核程序临界区
    ③其他需要完全屏蔽中断的原子操作过程
  4. 剥夺是针对进程来说的，抢占是针对作业来说的
    ①非剥夺是调度：非抢占式。指一个进程正在处理机上运行时，即使有某个更重或紧迫的进程进入就绪队列，依然让正在执行的进程继续执行，直到进程完成或发生某种事件进入阻塞状态时，才把处理机分配给更为重要或紧迫的进程。即处理机用完才给
    ②剥夺式调度：当一个进程正在处理机上执行时，若有某个更为重要或紧迫的进程需要使用处理机，则立即暂停正在执行的进程，将处理机分配给这个更为重要或紧迫的进程。要遵循一定的原则：优先权、短进程优先、时间片原则。所以时间片轮转是抢占式
  5. 调度的基本准则：（我认为也是整个操作系统的一些资源衡量指标）
    ①CPU利用率：CPU是计算机系统中最重要的资源，要是使他一直处于忙的状态
    ②系统吞吐量：单位时间内CPU完成的作业数。长作业消耗较长的处理机时间，降低吞吐量
    ③周转时间：从作业提交到作业完成所经历的时间，包括作业等待、在就绪队列中排队、处理机上运行以及输入输出所花费的时间总和
    常用公式：
    周转时间 = 作业完成-作业提交时间
    平均周转时间是指多个作业周转时间的平均值
    带权周转时间是指作业周转时间（从进入等待到完成）与作业实际运行时间的比值（获得处理机的时间）：

    ④等待时间：在就绪队列中等待的时间，处理机调度算法不影响作业执行或输入
    输出操作的时间，只影响作业在就绪队列中等待所花的时间。衡量一个调度算法优劣只需简单地考虑等待时间
    ⑤响应时间：从用户提交到请求到系统首次产生响应所用的时间。（和等待时间有区别，这里是首次响应的时间）
  6. 典型的调度算法
    ①先来先服务：适用于作业调度和进程调度，不可剥夺，效率低，对长作业有利，对短作业有利，有利于CPU繁忙型作业，不利于I/O繁忙型作业
    ②短作业优先：从后备队列中选择一个或若干个估计运行时间最短的作业，将他们调入内存运行。长作业运转周期会增加，可能导致长作业长期不被调度（饥饿现象-系统循环等待，与死锁有区别，死锁是调度策略问题），未考虑作业的紧迫程度，平均等待时间、平均周转时间最少
    ③优先级调度：优先级调度算法每次从就绪队列中选择优先级最高的进程，将处理机分配给他
    - 非剥夺式优先级调度算法
- 剥夺式优先级调度算法
    根据进程创建后其优先级是否可以改变，将进程优先级分为
    - 静态优先级：优先级不变
- 动态优先级：根据情况动态调整，主要依据进程占有CPU时间长短，就绪进程和等待CPU时间的长短
    ④高响应比优先调度：对先来先服务和短作业优先算法的一种综合平衡，计算响应比，选出响应比最高的作业投入运行

    由公式可知：等待时间相同时，要求服务时间越短，响应比越高，有利于短作业；要求服务时间相同时，等待时间越长，响应比越高，先来先服务；
    对于长作业，作业响应比可以随着等待时间的增加而提高，等待时间足够长后其响应比也提高了
    ⑤时间片轮转调度：适用于分时系统，总选择就绪队列第一个进程执行，先来先服务，剥夺式，运行完一个时间片及时没运行完也调度
    时间片轮转调度算法，**时间片的长度很重要**，**时间片足够大**，则所有进程在一个时间片内完成，则时间片算法退化成**先来先服务**，时间片很小，则会频繁调度
    ⑥多级反馈队列调度算法：时间片轮转调度和优先级调度的综合；通过动态调整进程优先级和时间片的大小

    设置多个就绪队列，每个队列的优先级不同，时间片长度也不同；仅当第一级队列为空时才执行第二级队列中的进程；如果正在执行i级队列，但是i-1级优先级较高的队列又进入新的进程，则此时新的进程将抢占正在运行进程的处理机，调度程序把正在运行的程序放回到第i级队列的末尾
